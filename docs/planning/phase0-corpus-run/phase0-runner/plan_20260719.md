## Phase 1 — Disposition model + ledger (pure, cross-platform)

- **Goal:** the data model for a run: instance dispositions, per-status turn counts, flagged /
  addable / unaddable counts, UNVERIFIED turns bucketed by cause. Pure dataclasses + a
  serialize/deserialize round-trip. No verify, no add — just the ledger the later phases fill.
- **Prerequisites:** none.
- **Files:** `src/belay/phase0/ledger.py`, `tests/test_phase0_ledger.py`.
- **Design:**
  - `Disposition(Enum)`: `VERIFIED_CLEAN`, `VERIFIED_FLAGGED`, `NO_VERIFIABLE_TURNS`, `ERRORED`
    (str-mixin, serializes as name — mirror `verdict.Status` at `verdict.py:34`).
  - `InstanceRecord(frozen)`: `trace_id`, `disposition`, `turn_status_counts: dict[str,int]`
    (keys are `Status` names), `flagged_turns: list[int]`, `flagged_addable: list[int]`,
    `flagged_unaddable: list[dict]` (turn index + cause), `unverified_causes: dict[str,int]`,
    `error: Optional[str]`.
  - `RunLedger(frozen)`: `instances: list[InstanceRecord]`, plus derived helpers computed as
    methods (NOT stored, so they can't drift): `violation_denominator()` = count of
    `VERIFIED_CLEAN`+`VERIFIED_FLAGGED`; `violating_instances()` = count `VERIFIED_FLAGGED`;
    `no_verifiable_count()`, `errored_count()`; `total_turns()`, `fail_turns()`;
    `unverified_by_cause()` (merged across instances).
  - `to_json(ledger) -> dict` / `from_json(dict) -> RunLedger`, fail-closed on unknown enum
    values (raise a named `ValueError`, never coerce — mirror `case.py`'s loader discipline).
- **Validation:** `uv run pytest tests/test_phase0_ledger.py`
  - RED tests: construct records of each disposition; assert the four derived counts; assert
    `violation_denominator()` excludes `NO_VERIFIABLE_TURNS` and `ERRORED`; assert JSON round-trip
    is identity; assert `from_json` on an unknown disposition raises.
- **Commit:** `Add the phase0 run ledger: dispositions + denominator-disciplined counts`

## Phase 2 — Report renderer + instrument-failure guard (pure, cross-platform)

- **Goal:** turn a `RunLedger` (+ a `Metrics` from `corpus.metrics.score`) into the published
  number, with the honesty guards as teeth.
- **Prerequisites:** Phase 1.
- **Files:** `src/belay/phase0/report.py`, `tests/test_phase0_report.py`.
- **Design:**
  - `violation_rate(ledger) -> Optional[float]`: `violating / denominator`, using a local
    `_ratio` (0-denominator ⇒ `None` = "n/a", never 0.0-that-reads-as-clean — copy the honesty
    of `metrics._ratio` at `metrics.py:105`).
  - `instrument_suspect(ledger, *, threshold) -> bool`: True when `NO_VERIFIABLE_TURNS`
    (+ errored) dominate the run (e.g. `no_verifiable + errored >= denominator`, or denominator
    == 0 with instances present). This is the R6 false-zero defense.
  - `render_report(ledger, metrics) -> str`: human text with, in order: run size + dispositions;
    the headline violation rate **with its denominator stated**, OR — when `instrument_suspect` —
    a loud "INSTRUMENT SUSPECT: traces captured ~no verifiable turns; this is UNVERIFIED-of-the-
    experiment, NOT a 0% violation rate" block *instead of* a headline rate; per-turn FAIL rate;
    UNVERIFIED rate by named cause; FP-rate = `1 - metrics.precision` (or "n/a" when precision is
    None); the flagged-unaddable count noted as "counted as violations, not replayable cases".
  - Never renders UNVERIFIED as PASS; never prints a rate without its denominator.
- **Validation:** `uv run pytest tests/test_phase0_report.py`
  - RED teeth: (a) a ledger of all-`NO_VERIFIABLE_TURNS` renders the INSTRUMENT SUSPECT block and
    does NOT print "0%" as a violation rate; (b) a mixed ledger prints `flagged/denominator` with
    the denominator visible; (c) 0-denominator ⇒ "n/a"; (d) UNVERIFIED appears only in its own
    section with causes, never in the violation numerator; (e) FP-rate matches `1 - precision`.
- **Commit:** `Add the phase0 report: violation rate + instrument-suspect guard + FP/UNVERIFIED rates`

## Phase 3 — Batch runner over traces, with an injectable verifier seam

- **Goal:** the driver that walks a trace directory, verifies each turn, classifies the
  disposition, ingests flagged turns, and builds the `RunLedger`. Testable cross-platform by
  injecting a fake verifier; wired to real `verify_turn` + `add_case` by default.
- **Prerequisites:** Phases 1–2.
- **Files:** `src/belay/phase0/runner.py`, `tests/test_phase0_runner.py`.
- **Design:**
  - `run_batch(trace_dir, *, corpus_dir, manifest_root, server_command, invariants, replays,
    timeout, captured_at, verifier=verify_turn, ingester=add_case) -> RunLedger`.
    - Enumerate `trace_dir.glob("trace-*.jsonl")` (stable sorted).
    - Per trace: `read_trace(path).records` → `tool_calls(derive_correlation(records))`. Resolve
      the trace's `.manifests` sibling (name convention from the snapshot dir — accept a
      `manifest_root` or derive per-trace; confirm against how the mint lays them out).
    - Per turn `n`: call `verifier(records, n, server_command=..., manifest_dir=...,
      invariants=..., replays=..., timeout=...)` → `TurnVerdict`. Tally `turn_status_counts`;
      collect `cause` for UNVERIFIED turns (already a canonical bucket, `turn.py:126`).
    - **Disposition rule:** any FAIL turn ⇒ `VERIFIED_FLAGGED`; else ≥1 turn with status in
      {PASS, WARN, FAIL} that actually replayed ⇒ `VERIFIED_CLEAN`; a trace with 0 `tools/call`
      turns, or all turns UNVERIFIED ⇒ `NO_VERIFIABLE_TURNS`; an exception during read/verify ⇒
      `ERRORED` (record the message; never let one bad trace abort the batch).
    - **Flag + ingest:** for each FAIL turn, attempt `ingester(corpus_dir, records=...,
      target_turn_index=n, verdict=..., manifest_dir=..., server_command=..., invariants=...,
      human_label="pending", replays=..., timeout=..., source_trace_id=..., captured_at=...)`.
      On success → `flagged_addable`. On the named `ValueError` add_case raises for an
      unrestorable pre-state (`add.py:122-133`) → `flagged_unaddable` with the cause string;
      the turn STAYS in the violation numerator (the instance is still `VERIFIED_FLAGGED`).
  - The seam: default `verifier=verify_turn`, `ingester=add_case`; tests pass fakes returning
    canned `TurnVerdict`s and recording `add_case` calls — so all arithmetic is exercised with no
    Seatbelt and no replay.
- **Validation:** `uv run pytest tests/test_phase0_runner.py`
  - RED tests (fake verifier + synthetic traces built with `conftest.trace_of` / the
    `test_corpus_add._synthetic_run` pattern):
    1. one flagged + one clean + one all-UNVERIFIED + one that raises ⇒ each maps to the right
       disposition; denominator excludes the last two.
    2. a FAIL turn with a restorable synthetic pre-state ⇒ `add_case` called, `flagged_addable`.
    3. a FAIL turn whose pre-state is unrestorable (fake ingester raises the named ValueError) ⇒
       `flagged_unaddable`, still counted as a violation, batch continues.
    4. one bad trace file (corrupt line) ⇒ that instance `ERRORED`, others still processed.
    5. UNVERIFIED causes tallied by bucket.
- **Commit:** `Add the phase0 batch runner over traces, verifier/ingester seam, disposition rules`

## Phase 4 — CLI: `belay phase0 run` + `belay phase0 report`

- **Goal:** wire the runner + report into the `belay` CLI, mirroring the corpus subparser pattern
  (`cli.py:1090`).
- **Prerequisites:** Phases 1–3.
- **Files:** `src/belay/cli.py` (new `_cmd_phase0_run`, `_cmd_phase0_report`, subparser block),
  `tests/test_phase0_runner.py` (CLI-level cases) or a small `tests/test_phase0_cli.py`.
- **Design:**
  - `belay phase0 run <trace-dir> --manifest-root DIR --corpus-dir corpus/local --ledger OUT.json
    [--invariants FILE] [--no-default-invariants] [--replays 3] [--server -- cmd...]`.
    Reads the clock ONCE at the boundary for `captured_at` (the only clock read; injected into
    `run_batch`), applies the same effective A1 policy as `verify`/`corpus add`
    (`default_invariants()` unless `--no-default-invariants`, plus `--invariants` file; fail-closed
    on a malformed file → exit 2), writes the ledger JSON, prints the report. Exit non-zero only on
    a hard error, not on "violations found" (this is a measurement, not a gate check).
  - `belay phase0 report <ledger.json> [--corpus-dir corpus/local]`: loads the ledger, runs
    `score()` over the corpus cases, prints `render_report`. Pure re-render, no replay.
  - Ledger default output path under gitignored run output (e.g. `runs/` — already reserved in
    `.gitignore:19`); never commit real run data.
- **Validation:** `uv run pytest tests/test_phase0_cli.py`
  - RED: `phase0 run` over a synthetic trace dir with an injected/stubbed verifier path (or the
    darwin e2e in Phase 5) writes a ledger and prints a denominator-bearing report; a malformed
    `--invariants` file exits 2; `phase0 report` re-renders a fixture ledger deterministically.
- **Commit:** `Add belay phase0 run/report CLI over the batch runner`

## Testing Strategy

- **Unit/pure (cross-platform, CI):** ledger arithmetic, report rendering + instrument guard,
  runner disposition logic via the fake verifier/ingester seam and the synthetic-flagged-run
  pattern (`test_corpus_add.py:109` region: fake `present` handle + synthetic manifest + tree).
- **Integration (darwin-gated):** one real capture→verify→flag→ingest e2e, `skipif` off-darwin
  (honest, mirrors `test_launch_demo`/`test_corpus_add` darwin gating).
- **No network, no LLM, no clock in arithmetic.** The single clock read is at the CLI boundary and
  injected as `captured_at`. Determinism is asserted (fixed inputs ⇒ identical ledger+report).
- **Honesty teeth (must watch them fail first):** all-empty mint must NOT render as clean 0%
  (instrument guard); UNVERIFIED must land in neither the violation numerator nor the clean bucket;
  a rate with a 0 denominator must be "n/a", never 1.0/0%.
- Full suite green after each phase: `uv run pytest`.

## Environment & Determinism

- Env vars are only relevant to the *live capture* (`BELAY_TRACE_DIR`, `BELAY_SANDBOX_SCOPE`,
  `BELAY_SNAPSHOT_DIR`) — exercised only in the darwin e2e. The runner/report themselves read no
  env and no network.
- macOS-only for the e2e and any real replay; off-darwin those tests skip and say so. The pure
  phases run everywhere.

## Edge Cases & Error Handling (with the verdict each produces)

- **Trace with 0 `tools/call` turns / handshake-only** ⇒ `NO_VERIFIABLE_TURNS`, excluded from the
  denominator, feeds the instrument-suspect guard. (R6 false-zero defense.)
- **All turns UNVERIFIED** (batching/unrestorable, R7) ⇒ `NO_VERIFIABLE_TURNS`; causes tallied.
- **FAIL turn, unrestorable pre-state** ⇒ `flagged_unaddable`, still a violation, not a corpus case.
- **Corrupt trace line / read error** ⇒ that instance `ERRORED` (message recorded), batch
  continues; never a silent drop, never a fabricated verdict.
- **Malformed `--invariants` file** ⇒ fail-closed exit 2 (mirror `verify`/`corpus add`).
- **0-denominator report** ⇒ violation rate "n/a"; if instances exist ⇒ instrument-suspect.
- **UNVERIFIED anywhere** ⇒ counted as UNVERIFIED with a named cause; never PASS, never a violation.
