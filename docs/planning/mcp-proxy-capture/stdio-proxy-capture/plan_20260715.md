### Phase 2 — 🟢 The raw streaming pump (GREEN)

**Goal:** make Phase 1 pass. M1, M2, M3, M10, M13.

- **Design (adopted from `mcpsnoop`'s audited source):** **forward by streaming chunks; observe by
  peeking a bounded copy.** The data path is **never frame-reassembled** — byte-exactness becomes a
  *structural property*, not something we test for and hope. Cap the observed copy (`MAX_FRAME =
  16 MiB`); the data path stays unbounded.
- **Two independent async pumps**, message-type-agnostic (M3). **Not** a request/response state
  machine — a server-originated `sampling/createMessage` deadlocks one.
- **Forward `stderr`** unmodified (spec: servers MAY log anything there; clients SHOULD NOT read it
  as failure).
- **Never rewrite** `protocolVersion`, capabilities, `_meta`, `instructions`. Never inject.
- **Shutdown:** client closes our stdin → we close the server's stdin → wait → SIGTERM → SIGKILL.
  **Never orphan the child.**
- **Files:** `src/belay/proxy.py`, `src/belay/__main__.py` (thin `belay proxy` entry)
- **Validation:** `uv run pytest` → Phase 1 test **passes**
- **Commit:** `Add byte-transparent stdio MCP proxy (streaming forward, bounded peek)`

### Phase 4 — Trace format + writer

**Goal:** the interchange format. **M5, M6, M12, M14, M15. A4, A7, A9, A10.**

- **RED first:** `tests/test_trace_format.py` — schema round-trip (A7); per-frame hash stability
  across two identical runs (A4); **byte-exact round-trip including a non-UTF-8 frame** (A9);
  **owner-only file mode** (A10).
- **Format:** append-only **JSONL**, one frame per line. `raw` is **base64** (a non-conforming server
  can emit bytes that are not a valid JSON string at all — and M11 says we record those honestly
  rather than crash). Human-readability is **C7's** job; add `belay trace cat` whenever wanted.
- **Hashes:** `hash_raw` (bytes as received) **and** `hash_canonical` + `canonical_form`.
  **Never** canonicalize→hash→discard the original.
- **Timing:** `t_in` / `t_out` / `observation_point: "proxy"`, **outside the hashed content**
  (otherwise A4 can never pass) and never implying server-side timing.
- **Extensible record kinds** (`kind: "frame"`) + a documented rule for unknown kinds / higher
  versions. C2 appends `denial`, C3 appends `nondeterminism`. **No migration framework.**
- **State-handle slot (S2):** three-state, `{"status": "absent"}`, **left empty**.
  > If it can only express present/absent, C2 must break the schema or overload "absent" to mean
  > both *"C1-era trace"* and *"C2 tried and failed"* — **that overload is how a false PASS is born.**
- **M15:** owner-only modes (`0o600` / dir `0o700`); trace sensitivity documented prominently.
  **No capture-time redaction — ever.**
- **Files:** `src/belay/trace.py`, `src/belay/hashing.py`, `docs/technical/TRACE_FORMAT.md`,
  `tests/test_trace_format.py`
- **Open decision (this phase):** canonical form = JCS (RFC 8785) or `belay/jcs-v1`. **Document and
  version it either way. Must not block A1.**
- **Validation:** `uv run pytest` → all pass
- **Commit:** `Add versioned append-only JSONL trace format with dual hashing`

### Phase 5 — Frame indexing + correlation

**Goal:** M4, and `tools/call` as a **derived index**, not the unit of capture.

- **RED:** correlation test — client `id:1` and server `id:1` **must not** cross-wire.
- **Key on `(direction, id)`** — the two id spaces are separate namespaces and may both start at 1.
  Responses carry no `method`, so correlate by id.
- **Side-table only.** Writing into `_meta` would break byte-neutrality.
- **Named cause required:** a frame exceeding the observe cap **forwards exactly** but may index
  incompletely → record the cause; **never silently index as absent** (this is where UNVERIFIED's
  "named cause" contract is born).
- **Requests that never get a response are legal** (cancellation is racy by design) — the writer must
  **not** leak entries or block awaiting one.
- **Files:** `src/belay/index.py`, `tests/test_correlation.py`
- **Validation:** `uv run pytest`
- **Commit:** `Correlate frames on (direction, id) with named-cause indexing gaps`

### Phase 6 — Annotation snapshot (tri-state) + re-snapshot

**Goal:** M7, M8, S1. **A5.** *The field C5's zero-config invariants are built from.*

- **RED:** `tests/test_annotations.py::test_not_declared_is_distinguishable_from_declared_false`.
- **Tri-state per annotation:** `declared-true` | `declared-false` | `not-declared`, **plus** a
  separate fact for *"this tool had no `annotations` object at all."*
  > **Spec defaults are `readOnlyHint:false`, `destructiveHint:**true**`, `idempotentHint:false`,
  > `openWorldHint:**true**` — fail-safe, but a default is NOT a declaration.** An un-annotated tool
  > defaulted to `readOnlyHint:false` lets C4 reason *"it mutated, it never claimed read-only,
  > therefore fine → PASS"* — **a false PASS manufactured by a default**, which `:193-194` and
  > `:205-206` forbid. **Record what was declared. Never materialize a default into the trace.**
- **S1:** record **incoherence** (`readOnlyHint:true` + `destructiveHint:true`) as a fact —
  `destructiveHint`/`idempotentHint` are *"meaningful only when `readOnlyHint == false`"*.
- **M8:** re-snapshot on `notifications/tools/list_changed`; snapshots are **timestamped and
  appended, never overwritten**. A single snapshot verifies against a **stale contract**.
- **Files:** `src/belay/annotations.py`, `tests/test_annotations.py`
- **Validation:** `uv run pytest`
- **Commit:** `Snapshot tool annotations with mandatory tri-state and re-snapshot on change`

### Phase 7 — Per-call session context + honest error recording

**Goal:** M9, M11, S4. **A6.**

- **M9 — RED:** a frame carries **resolved** `protocol_version` + client identity, reconstructable
  from **either** a handshake (≤2025-11-25) **or** per-request `_meta` (≥2026-07-28).
  > **The 2026-07-28 revision ships in 13 days** and removes `initialize`/`initialized` (SEP-2575)
  > and the protocol session (SEP-2567). *"Session start" has 13 days to live.* Per-call resolution
  > costs ~nothing under raw forwarding and keeps the migration off C2/C3's critical path.
- **M11 — RED:** malformed / non-conforming server → **recorded error, never a dropped turn**;
  **protocol error** (`error` member, no `result`) vs **tool execution error** (success envelope,
  `isError:true`) **distinguishable**.
  > **2025-11-25 moved the line** (SEP-1303: input-validation errors became execution errors). The
  > same failure has different shapes across revisions. **Record raw; derive classification. Never
  > conflate.** Borrow `mcp-vcr`'s named-cause pattern (`timeout|server_crash|pipe_error|
  > malformed_response`).
- **S4:** session-window facts (proxy live T0→T1) so **R6's denominator is expressible**.
- **Files:** `src/belay/session.py`, `tests/test_session_context.py`, `tests/test_errors.py`
- **Validation:** `uv run pytest`
- **Commit:** `Record per-call session context and honest, classified errors`

## 4. Environment & determinism

- `BELAY_TRACE_DIR` — artifact root, default `./traces/` (gitignored). **Configurable on day 1**: the
  Phase-0 corpus (≥50 runs) must not live in the source checkout.
- **Pin the interpreter** (`.python-version`, `requires-python=">=3.10"`) — the SDK dev dep needs
  ≥3.10; system Python is 3.9.
- **No network in any test.** No clock-dependent or random-ordering assertions.
- **Timing is excluded from hashed content** — otherwise A4 (stable hashes) can never pass.
- **A test whose result depends on clock, network, or random ordering is a bug in this plan.**

## 5. Edge cases & error handling — and what each records

**C1 emits no verdict.** But every failure must land as a **named, recorded cause**, because
downstream UNVERIFIED must point at one and **C1's capture is the only place it can originate**.

| Edge case | Behavior |
|---|---|
| Malformed / non-conforming server line | **Record honestly, forward as-is.** Never drop the turn. Named cause. |
| Non-UTF-8 bytes | Round-trip via base64. **Must not crash the writer** (A9). |
| Batch array (invalid ≥2025-06-18) | **Forward it**, record, flag non-conformant. **Neutral beats conformant.** |
| Server-originated request (`sampling/createMessage`, `roots/list`, `elicitation/create`, `ping`) | Forward. A request/response loop would **deadlock** — the most likely naive failure. |
| Progress notification | **Forward immediately, unbuffered** — they reset client timeout clocks; buffering causes timeouts on calls that would otherwise succeed. |
| Cancelled request (never answered) | **Legal.** Writer must not leak an entry or block. |
| Frame exceeds observe cap | **Forwards exactly**; index records a **named cause**. Never silently absent. |
| Server writes to stderr | **Forward.** Spec: MAY be any logging; clients SHOULD NOT read it as failure. |
| `tools/list` never arrives | Recorded as a named cause. Annotations are **`not-declared`**, never defaulted. |
| Client closes stdin | Close server stdin → wait → SIGTERM → SIGKILL. **Never orphan the child.** |
| `structuredContent` duplicated into a text block | Record **both** faithfully (S3); downstream must not double-count. |

## 6. Agent execution notes

**Checkpoints for human review:**
1. **After Phase 1** — the differential is C1's whole thesis. **Review it before making it pass.** A
   wrong gate makes every later green meaningless.
2. **After Phase 4** — the trace schema is the interchange format for the whole engine and the
   hardest thing to change later.
3. **Before Phase 9** — the coverage statement is a public honesty claim.

**Likely to need iteration:** the streaming pump's chunk boundaries vs. the peek buffer (Phase 2);
subprocess shutdown races (Phase 2); the canonical-form choice (Phase 4).

**Ask before proceeding if:** the canonical form materially changes what "divergence" means for C4;
any acceptance test looks like it needs a timing assertion (it doesn't — that would assert something
false); or a phase seems to need snapshot/restore (**it doesn't — that's C2**).

**Parallelisable:** Phases 4/5 are largely independent of 6/7 once the trace writer's interface
exists. Phases 1–3 are **strictly sequential** and must not be parallelised — they are the gate.

**Guardrail check for executing agents:** if a task drifts toward Belay *orchestrating* the agent, or
toward emitting **any** verdict, **stop and flag**. C1 records; it does not judge.
