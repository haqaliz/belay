## Phase 1 — The invariant format + loader (RED first)
**Goal:** M1 + A7 (operator-only). The declaration and its provenance boundary.
- **RED:** `tests/test_invariants.py` — load a `{scope, rule}` list from a JSON file; malformed file ->
  clean error (not a crash); a trace with an "invariant"-shaped record is NOT read as policy.
- **GREEN:** `src/belay/verify/invariants.py` — `Invariant(scope: bytes, rule: str)`,
  `load_invariants(path) -> list[Invariant]` (JSON, stdlib). Scope stored as RAW BYTES (byte-prefix
  discipline). rule in {"read-only"} for v0 (+ no-create/no-delete if S2).
  🔴 **A7:** the loader reads ONLY the operator file. Assert nothing reads an invariant from the trace —
  the trace is evidence, the invariant is policy. A test constructs a trace carrying a permissive
  "invariant" record and asserts A1 does NOT honor it.
- **Commit:** `Add the invariant declaration format and an operator-file loader`

## Phase 2 — The evaluator, grounded on the delta (RED first)
**Goal:** M2 + M4 + A6 + A4. The verdict, and its honest bounds.
- **RED:** `tests/test_invariant_eval.py`
  - A6: a `read-only` invariant `{scope:"tests/"}` + a delta touching `tests/test_auth.py` -> FAIL
    naming the invariant + the byte path. A path OUTSIDE scope (e.g. `src/x`) -> PASS. **Byte-prefix**,
    so `tests/` matches `tests/test_auth.py`; assert a near-miss (`testsuite/x`) does NOT match.
  - A4: `delta is None` -> UNVERIFIED (never PASS). A network invariant (`no-egress`) -> UNVERIFIED
    (mirror openWorldHint; never PASS, never fabricated FAIL). **Positive control:** a real filesystem
    violation still FAILs, so UNVERIFIED isn't swallowing everything.
- **GREEN:** `invariants.py` — `evaluate_invariant(inv, delta, turn_index, ...) -> Verdict(axis="A1",
  kind="invariant")`. Reuse `effect._paths(delta)`. read-only: any touched path under scope (byte-
  prefix) -> FAIL. delta None / network -> UNVERIFIED. Message names invariant, turn, violating path(s).
- **Commit:** `Evaluate invariants against the replay delta, network UNVERIFIED-only`

## Notes for executing agents
- **A1 is TASK-SCOPED and TOOL-INDEPENDENT.** "was the task policy violated by the observed effect?" —
  NOT "did the tool honour its own hint?" (that is C4). If your A1 verdict equals C4 effect-conformance
  on the demo turn, you built it wrong.
- **Invariants are an OPERATOR FILE, never read from the trace.** The trace is evidence; the invariant
  is policy. A run must not author its own passing invariant.
- **A1 evaluates reply.delta, never prose or a static lint.** Grounded by construction is the claim.
- **The demo targets the WEAKENING turn** (delta contains the tests/ write), not a later reading turn
  (delta empty, nothing to catch).
- **Network invariants are UNVERIFIED-only.** Never a network PASS, never a fabricated FAIL.
- **delta is None -> UNVERIFIED, never PASS.** Zero LLM. C1-C4's 348 + differential/teeth/import-guard
  pass UNTOUCHED.
