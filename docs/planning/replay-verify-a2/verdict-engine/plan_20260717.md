## Phase 1 — The Verdict model + the reduction (RED first)
**Goal:** M1 + M2 + A7. The verdict contract for the whole engine. Built first because everything
returns one.
- **RED:** `tests/test_verdict.py` — construct verdicts; assert the reduction.
- **A7 the structural guard:** `reduce([PASS, UNVERIFIED]) == UNVERIFIED`; `reduce([PASS, FAIL]) ==
  FAIL`; `reduce([UNVERIFIED, FAIL]) == FAIL`; `reduce([PASS, PASS]) == PASS`; `reduce([]) ==
  UNVERIFIED` (nothing checked is not a pass). **UNVERIFIED must rank ABOVE PASS/WARN.** Watch a
  wrong ordering (UNVERIFIED below PASS) fail this.
- **A1-ready:** the reduction takes N verdicts across axes; a test reduces a fake A1 FAIL + an A2 PASS
  -> FAIL, proving A1 slots in unchanged.
- **GREEN:** `src/belay/verify/verdict.py` — `Status` (PASS/WARN/FAIL/UNVERIFIED), `Verdict(axis, kind,
  status, observed, expected, message)`, `reduce(verdicts) -> Status`.
- **Commit:** `Add the Verdict model and the worst-wins reduction (UNVERIFIED never PASS)`

## Phase 2 — Result-equivalence, gated on determinism (RED first)
**Goal:** M3 + A1 + A2 + A3. The trace-infidelity check. **The determinism gate is the point.**
- **RED:** `tests/test_verify_result.py`
  - A2: a clean deterministic turn -> PASS.
  - A1: a fabricated/divergent result on a DETERMINISTIC tool -> FAIL, message carries the exact
    recorded-vs-observed diff. **Watch it fail against a stub that PASSes everything.**
  - A3: a clock/random tool that diverges -> UNVERIFIED, **not FAIL**. Watch a naive (no-gate)
    implementation FAIL this.
  - Malformed replayed reply (unparseable) -> a DISTINCT message from a value mismatch.
- **GREEN:** `src/belay/verify/result.py`
  - Read `engine.replay_turn`'s `TurnReply`. EQUAL -> PASS (no determinism call). DIVERGED -> run
    `determinism.classify_determinism`; DETERMINISTIC -> FAIL (diff), NONDETERMINISTIC -> UNVERIFIED
    (carry axis). `result_equivalence is None` -> UNVERIFIED.
  - **Only classify on DIVERGED** (cost). Build the diff from recorded_reply + replayed_reply.
- **Commit:** `Verify result-equivalence, gated on determinism so nondeterminism is never a FAIL`

## Checkpoints
1. **After Phase 1** — the Verdict model is the whole engine's verdict contract. Review the reduction
   (UNVERIFIED>PASS) before everything depends on it.
2. **After Phase 2** — the determinism gate is the single most likely defect. Review A3 (clock->
   UNVERIFIED) with its positive control (real infidelity->FAIL).
3. **After Phase 5** — A9 (PASS-on-the-cheat) is the non-redundancy claim; A8 (zero-LLM) is the
   positioning. Both are correctness, not nicety.
4. **After Phase 6** — the network scope decision (build vs honest-UNVERIFIED). This is where
   over-claiming is easiest.

## Notes for executing agents
- **C4 emits a verdict, but A2 catches INFIDELITY not CHEATING.** PASS = "the trace reproduces", never
  "the agent did right". Say exactly that in every message; never more.
- **Zero LLM. No model import anywhere in src/belay/verify.** A8 enforces it.
- **UNVERIFIED never PASS** is the reduction's structure, not a convention.
- **Never let a nondeterministic divergence become a FAIL** — the determinism gate is mandatory.
- **NOT_VERIFIABLE must not fall through to PASS.**
- C1+C2+C3's 290 tests + differential/teeth/import-guard pass UNTOUCHED.
