Five phases, sequential (each builds on the last). Phases 1–4 are the must; Phase 5 is optional/deferable.
Every phase is RED-first; the anti-vacuity and honesty controls are named per phase — watch them fail.

## Global constraints (bind every task; copy verbatim into reviewer prompts)
- **No new verdict axis, no LLM, no egress.** C6 stores/regresses the existing A1/A2 verdicts. Any
  auto-labeler is an LLM judge and is FORBIDDEN — the human label is an explicit input, never inferred.
- **UNVERIFIED is never PASS and never a positive.** Excluded from precision/recall; reported on its own
  coverage line. A regression (verdict changed) is DISTINCT from a SKIP (UNVERIFIED-by-substrate/server).
- **The label-trap is the #1 risk:** an engine-authored/`pending` label must be UNCOUNTABLE in P/R (a
  watched test). Precision-100%-by-construction destroys the metric.
- **Determinism:** no clock/network/random in engine paths. Any timestamp is PASSED IN, never read from
  the clock inside library code (tests may stamp). Reuse `verify_turn` unchanged for recompute.
- **Cases live under `corpus/local/`** (gitignored, `.gitignore:19`), owner-only, never committed.
- **C1–C5 untouched:** do not modify `verify_turn` composition, `evaluate_invariant`,
  `render_effect_verdict`, `engine.py` replay logic, or any C1–C5 test. The ONE allowed engine change is
  M2 (`load_snapshot` relative-tree resolution), backward-compatible.

## Phase 1 — Case format + `load_snapshot` case-relative tree (M1, M2). FOUNDATION.
**Goal:** the on-disk case format and the minimal engine change that lets a case bundle its own pre-state.
- **New module:** `src/belay/corpus/case.py` — a `Case` dataclass + `load_case(dir)` / `write_case(dir, case)`.
  `case.json` fields (M1): `id`, `target_turn_index`, `expected` (the per-sub-verdict set: a list of
  `{axis, kind, status}` PLUS `reduced_status`), `human_label` ∈ {`true-positive`, `false-positive`,
  `unverifiable`, `pending`} (default `pending`), `invariants` (list of `{scope, rule}`), `server_command`
  (list[str]), `replays` (int), `timeout` (float), `provenance` (`{source_trace_id, captured_at}` — both
  passed in), `capture_platform`, `capture_capabilities` (sorted list). Fail-closed on an unknown label or
  missing required field (named `ValueError`, mirroring `load_invariants`).
- **M2 engine change:** in `src/belay/replay/persist.py::load_snapshot`, resolve `tree_path`:
  if `Path(tree_path).is_absolute()` keep today's behavior; else resolve against `manifest_path.parent`.
  `persist_snapshot` is UNCHANGED (still writes absolute). Backward-compatible.
- **RED:** `tests/test_corpus_case.py` — (a) a Case round-trips through write/load byte-stably; (b) unknown
  label → ValueError; (c) `pending` is the default when omitted; (d) missing required field → ValueError.
  `tests/test_persist_relative_tree.py` — (e) an EXISTING absolute-`tree_path` manifest still loads to the
  same tree (backward-compat, watch it pass before AND after the change); (f) a manifest with a RELATIVE
  `tree_path` resolves against the manifest's own dir (watch it FAIL first — today it builds a bad
  relative Path).
- **GREEN:** the module + the resolution rule.
- **Commit:** `Add the corpus case format and case-relative snapshot tree resolution`

## Phase 2 — `belay corpus add` (M3).
**Goal:** compose a self-contained case from a flagged run.
- **New:** `src/belay/corpus/add.py::add_case(corpus_dir, *, records, target_turn_index, verdict,
  manifest_dir, server_command, invariants, human_label="pending", replays, timeout, source_trace_id,
  captured_at) -> Path`. It: mints `<case-id>`; writes `corpus/local/<case-id>/`; copies the trace slice
  to `trace.jsonl`; copies the turn's persisted manifest to `manifest.json` and the referenced tree into
  `prestate/`, rewriting `manifest.json`'s `tree_path` to the relative `"prestate"`; derives `expected`
  from `verdict` (the per-sub-verdict set + reduced status); stores policy/server_command/provenance;
  takes `human_label` as an EXPLICIT arg (default `pending`).
- **CLI:** a `corpus` subparser group in `cli.py` with `add` (mirror the `verify` subparser + `_cmd_verify`
  structure). `--label` optional (default pending). `belay corpus add <trace> --turn N --manifest-dir ...
  --server ... [--invariants FILE] [--label ...]` — recompute the verdict via `verify_turn`, then
  `add_case`.
- **RED:** `tests/test_corpus_add.py` — (a) add composes exactly the 4 artifacts; (b) the tree is COPIED
  (case survives deleting the original manifest_dir — assert load after rm); (c) `manifest.json.tree_path`
  is relative `"prestate"`; (d) 🔴 the ENGINE NEVER LABELS: `add_case` with no label → `human_label ==
  "pending"` (NOT derived from a FAIL verdict); assert a FAILing verdict still yields `pending`. (e)
  `expected` carries the per-sub-verdict axis/kind/status set, not just reduced status.
- **GREEN:** `add_case` + the CLI wiring.
- **Commit:** `Add belay corpus add — compose a self-contained case from a flagged run`

## Phase 3 — `belay corpus run` regression, SKIP distinct from regression (M4, M6).
**Goal:** re-verify each case and assert its expected verdict; a regression fails, a substrate/server gap
skips.
- **New:** `src/belay/corpus/run.py::run_case(case_dir) -> CaseResult` and `run_corpus(corpus_dir) ->
  CorpusRun`. `run_case`: `load_case`, `read_trace` the slice, point `verify_turn` at the case's
  `manifest.json` (via a manifest_dir it can consume) with the stored `server_command`, `invariants`,
  `replays`, `timeout`; recompute the per-sub-verdict set; classify: `MATCH` (equals expected),
  `REGRESSION` (a decisive verdict differs — name the case + diverging axis/kind), or `SKIP`
  (recompute is UNVERIFIED-by-substrate/missing-server/capability-mismatch — DISTINCT from a regression).
- **CLI:** `belay corpus run [corpus_dir]` — aggregate MATCH/REGRESSION/SKIP counts; exit non-zero iff any
  REGRESSION (a pure SKIP is not a failure). Render each regression with the diverging sub-verdict.
- **RED (portable, no replay — the comparison layer is pure):** `tests/test_corpus_run.py` — build a case
  whose stored `expected` set and an INJECTED recomputed set (a) match → MATCH; (b) differ on one axis →
  REGRESSION naming the case AND the axis/kind (watch it fail against a "compare reduced status only"
  stub, which would MISS an axis flip with unchanged reduced status — the D4 point); (c) recompute
  UNVERIFIED → SKIP, asserted DISTINCT from REGRESSION (watch a naive "any-non-match is a regression" stub
  fail this). Structure `run_case` so the comparison/classification is a pure function testable without
  spawning a server.
- **RED (real round-trip, darwin-gated `skipif` like `test_launch_demo.py`):** `tests/test_corpus_roundtrip.py`
  — a real flagged run (reuse the weakening-editor fixture) → `add_case` → `run_case` → MATCH (the A1 FAIL
  reproduces). This is acceptance bullet 1.
- **GREEN:** `run_case`/`run_corpus` + CLI.
- **Commit:** `Add belay corpus run — regression by re-verify, skip distinct from regression`

## Phase 4 — precision / recall / coverage metric (M5, M6). THE HONEST NUMBER.
**Goal:** a pure metric over (engine verdict, human label) pairs, UNVERIFIED handled honestly.
- **New:** `src/belay/corpus/metrics.py::score(cases) -> Metrics`. Positive = engine verdict FAIL. Over
  the subset {verdict ∈ FAIL/PASS} ∩ {label ∈ BAD(true-positive)/GOOD(false-positive)}: `TP/FP/FN/TN`,
  `precision = TP/(TP+FP)`, `recall = TP/(TP+FN)` (define 0-denominator as `None`/"n/a", never 0/0=1).
  UNVERIFIED verdicts and `unverifiable`/`pending` labels are EXCLUDED from P/R and counted on a separate
  `coverage` line (decisive-adjudicable / total-adjudicable), broken down by UNVERIFIED cause. Pure
  function — no `verify_turn`, no replay, no network, runs everywhere.
- **CLI:** `belay corpus score [corpus_dir]` (or fold into `corpus run --score`) prints precision, recall,
  AND coverage side by side (never P/R without coverage).
- **RED:** `tests/test_corpus_metrics.py` — (a) a fixture corpus with KNOWN labels+verdicts → P/R/coverage
  equal HAND-COMPUTED values (acceptance bullet 3); (b) 🔴 THE LABEL-TRAP: a case whose label is `pending`
  (or absent) is NOT counted in P/R (watch a "count every FAIL as TP" stub inflate precision to 1.0 and
  fail this); (c) 🔴 UNVERIFIED verdict is EXCLUDED from P/R and appears on the coverage line, NOT folded
  into PASS/FN (watch a "UNVERIFIED == PASS" stub fail this); (d) 0-denominator precision is "n/a", not 1.0.
- **GREEN:** `score` + CLI.
- **Commit:** `Add the precision/recall/coverage metric, UNVERIFIED excluded and coverage reported`

## Testing strategy
- **Pure tiers run everywhere:** case format, M2 resolution, `corpus run` comparison/classification, and
  ALL metric math. No replay, no network, no darwin.
- **Real round-trip is darwin/Seatbelt-gated** (`pytest.mark.skipif(sys.platform != "darwin", ...)`),
  reusing the weakening-editor fixture from `tests/fixtures/`.
- Keep `uv run pytest` green after every task (377 at start; net-add only).

## Edge cases → verdict
- Server unavailable / off-darwin / capability mismatch at `corpus run` → **SKIP (UNVERIFIED-by-substrate)**,
  never a regression, never a false pass.
- Missing case file / corrupt case.json → named error (fail-closed), not a silent skip.
- 0-denominator precision or recall → `None`/"n/a", never 1.0.
- `pending`/unlabeled case → excluded from P/R (counted only in the pending tally).
