  passed unchanged; frames sent verbatim (`:265-290`).**
- Frames decoded/forwarded: `engine._gather_frames` (`engine.py:386-412`, base64-decode
  `raw`).
- Result-equivalence: `engine.py:375` (`result_equivalence`), reply text compared.
- Delta: `engine.py:321-322` on the **scratch** (`result.workspace`).
- Seatbelt: reads globally unscoped (`seatbelt.py:244`), writes confined to scratch
  (`scope.py:137-144`, `launch.py:209`). So relocation must move BOTH the argv root and the
  in-root path args so reads/writes land in the scratch.
- The recorded root is now on `Manifest.source_root` (this aspect's input).

## The sharpened rule (from the shell determination)

- **Mutated arguments:** remap a string **iff its entire value is an absolute path under
  `source_root`** → swap prefix to scratch. Covers filesystem `path`; never touches
  `content`/`newText` (file content). Content-safe by construction.
- **Server argv:** remap a token that **is / is under** `source_root` → scratch.
- **Reply comparison:** substring-normalize BOTH roots → a placeholder, **comparison-only**
  (transient; never persisted, never written to scratch). Handles paths embedded in diffs.
- **Gating:** relocation runs only when `source_root` is present AND the turn carries an
  in-root absolute path (arg or argv). Else: exactly today's behavior.
- **Honest fallback:** in-root absolute path present but `source_root` absent → `UNVERIFIED`,
  named cause. Never a false verdict.
- **Shell (`command_line` embedded paths):** OUT — follow-up `replay-relocation-shell`.

## 2. Phases (each RED → GREEN → REFACTOR, suite green, one commit)

### Phase 1 — `relocate.py`: the pure primitives (no I/O, exhaustively unit-tested)

- **Goal:** the relocation logic as pure functions, so the delicate rules are tested in
  isolation before touching the replay flow.
- **Functions:**
  - `is_under(path_str, root) -> bool` — realpath-normalized prefix test; both normalized the
    same way (`os.path.realpath` semantics matching how scratch is resolved). Boundary-safe
    (`/a/b` is not under `/a/bc`).
  - `remap_prefix(path_str, from_root, to_root) -> str` — swap the `from_root` prefix to
    `to_root`; caller guarantees `is_under`.
  - `remap_argv(argv, from_root, to_root) -> (list, changed: bool)` — rewrite any token that
    is / is under `from_root`.
  - `remap_arguments(obj, from_root, to_root) -> (obj, changed: bool)` — recurse dict/list;
    for a **string** value, remap **iff the whole value `is_under(from_root)`**. Never
    substring. Returns a new structure; input untouched.
  - `canonicalize_reply(text, root_a, root_b, placeholder) -> str` — **substring** replace
    both roots with the placeholder (for comparison only). Order-independent; assert the two
    roots don't nest (disjoint by construction — one is the workspace, one a mkdtemp).
  - `turn_needs_relocation(frame_arguments, argv, root) -> bool` — true iff any in-root
    absolute path appears (drives gating).
- **Tests (write FIRST) in `tests/test_replay_relocate.py`:**
  `test_is_under_is_boundary_safe`, `test_remap_argv_only_in_root_tokens`,
  `test_remap_arguments_whole_value_path_only`,
  `test_remap_arguments_leaves_content_fields_untouched` (an `edit_file`-shaped dict:
  `{path: <in-root abs>, edits:[{newText: "text /that/looks/like/root but is content"}]}` —
  `path` remapped, `newText` untouched),
  `test_canonicalize_reply_is_symmetric_on_a_diff` (a unified-diff string with the abs path in
  `Index:`),
  `test_turn_needs_relocation_false_for_cwd_relative`,
  `test_remap_functions_are_pure` (inputs unmutated).
- **Commit:** `replay: pure path-relocation primitives (whole-value args, substring replies)`

### Phase 2 — the committed absolute-path fixture server

- **Goal:** an MCP server that takes an absolute root and addresses files by absolute path —
  the inverse of every existing fixture, so CI covers the class.
- **Design:** `tests/fixtures/abs_path_editor_server.py`, stdlib only, deterministic,
  newline-delimited JSON-RPC. Launched as `python abs_path_editor_server.py <ABS_ROOT>`.
  Tools:
  - `read_abs(path)` — reads the **absolute** path, returns content (deterministic).
  - `edit_abs(path, new_content)` — overwrites the absolute path; reply is a **fixed** string
    (or a diff containing the absolute path, to exercise reply normalization).
  Mirrors the honesty/determinism of `weakening_editor_server.py` but **absolute-path** by
  construction. Include a variant that writes a *benign* edit and one that writes a *corrupt*
  one (for FP vs FN).
- **Tests (write FIRST):** a direct smoke that the fixture speaks the protocol and uses
  absolute paths (so later e2e failures are unambiguous).
- **Commit:** `test: committed absolute-path MCP fixture server (the uncovered class)`

### Phase 3 — wire relocation into the replay flow (gated, additive)

- **Goal:** consume `source_root`, relocate argv + frames, normalize the reply, with the
  honest fallback — zero change when not relocating.
- **Design:**
  - Thread `source_root` from `Manifest` into `client.replay_turn` (via `engine.replay_turn`,
    which already loads the snapshot).
  - In `client.replay_turn` before `contained(...)` (`:322`): if relocating, `argv =
    remap_argv(server_command, source_root, resolved)`. (`resolved` = scratch root, available
    at `:323` — compute it before the spawn, or move the resolve up.)
  - In the frame resend path (`_gather_frames`/`converse`): for each `tools/call` frame,
    `remap_arguments` on the decoded JSON, re-serialize preserving `id`, forward. Keep the
    id-correlation (`_await_key`/`_matches`) working on the relocated frame.
  - In `result_equivalence` (`engine.py:375`): `canonicalize_reply` both recorded and
    replayed reply before comparison. **Comparison-only.**
  - **Gating + fallback:** compute `turn_needs_relocation`. If it needs relocation and
    `source_root is None` → return `UNVERIFIED` with cause `original workspace root not
    recorded; cannot relocate absolute paths`. If it doesn't need relocation → the existing
    path, byte-for-byte.
- **Tests (write FIRST):** the id round-trip on a relocated frame; the gating (a cwd-relative
  turn takes the unchanged path); the fallback UNVERIFIED.
- **Commit:** `replay: relocate absolute-path servers into the restored scratch (gated)`

## 4. Determinism & verdict impact

- **A2 only.** Makes A2's existing claim TRUE for absolute-path servers; changes no claim.
  A1/A3 untouched. `UNVERIFIED`-never-`PASS` strengthened (the fallback).
- Relocation is a pure function of (trace, snapshot, scratch-path); the scratch path is a
  mkdtemp but it is **canonicalized away** in every comparison (arguments remap is structural;
  reply canonicalize maps it to a placeholder), so the *verdict* is invariant to the specific
  scratch path — assert this.

## 5. Edge cases → verdict

| Case | Behavior |
|---|---|
| `source_root` present, in-root abs paths | relocate; verdict from restored scratch |
| `source_root` absent, in-root abs paths (old trace) | `UNVERIFIED` (named cause) — never guessed |
| No in-root abs paths (cwd-relative) | unchanged path, byte-for-byte |
| Out-of-root abs path (`/etc/...`) | not remapped; as today |
| Content string that looks like a path but isn't whole-value | not remapped (content-safe) |
| Reply path inside a diff | canonicalized for comparison only |
| scratch path nests with root | asserted disjoint; canonicalize is order-independent |

## 6. Agent execution notes

- **Sequence:** Phase 1 (pure) → Phase 2 (fixture) can run in **parallel** (disjoint files);
  Phase 3 depends on both; Phase 4 depends on 3.
- **Checkpoint after Phase 3** — the wiring into the moat is the risky change; confirm the
  cwd-relative suite is still byte-green before the e2e phase.
- **Guardrails:** never remap a non-whole-value string in arguments (content corruption);
  reply normalization is comparison-only (never persisted); if relocation can't establish a
  trustworthy prefix, `UNVERIFIED`, never a guess. If a fix seems to need changing a
  cwd-relative verdict, STOP — the gate is wrong.
- **Ask before proceeding if:** the frame re-serialization can't preserve byte-fidelity of
  non-path fields (it must — only path values change), or if `result_equivalence` compares
  structured JSON rather than text (adjust canonicalization accordingly).
